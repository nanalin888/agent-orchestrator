## Prerequisites

- Python 3.10 or higher
- OpenRouter API key (get one free at [openrouter.ai/keys](https://openrouter.ai/keys))

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/nanalin888/agent-orchestrator.git
    cd agent-orchestrator
    ```
  </Step>
  
  <Step title="Create virtual environment">
    ```bash
    python3 -m venv .venv
    source .venv/bin/activate  # On Windows: .venv\Scripts\activate
    ```
  </Step>
  
  <Step title="Install dependencies">
    ```bash
    pip install -e .
    ```
  </Step>
  
  <Step title="Configure environment">
    ```bash
    cp .env.example .env
    ```
    
    Edit `.env` and add your OpenRouter API key:
    ```
    OPENROUTER_API_KEY=sk-or-v1-your-key-here
    ```
  </Step>
  
  <Step title="Start the service">
    ```bash
    python3 -m src.main
    ```
    
    The server starts at `http://localhost:8000` with 12 agents loaded.
  </Step>
</Steps>

## Your First Agent Request

<CodeGroup>
  ```bash cURL
  curl -X POST http://localhost:8000/agents/fast-assistant/run \
    -H "Content-Type: application/json" \
    -d '{
      "messages": [
        {"role": "user", "content": "What is 2+2?"}
      ]
    }'
  ```
  
  ```python Python
  import httpx
  
  response = httpx.post(
      "http://localhost:8000/agents/fast-assistant/run",
      json={
          "messages": [
              {"role": "user", "content": "What is 2+2?"}
          ]
      }
  )
  print(response.json())
  ```
  
  ```javascript JavaScript
  const response = await fetch('http://localhost:8000/agents/fast-assistant/run', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      messages: [
        { role: 'user', content: 'What is 2+2?' }
      ]
    })
  });
  
  const data = await response.json();
  console.log(data);
  ```
</CodeGroup>

## Web UI

Open your browser and navigate to `http://localhost:8000`:

- **Chat Tab** — Interactive chat with any text agent
- **Video Tab** — Generate videos with Veo or Wan models
- **Dashboard Tab** — Real-time model availability status
